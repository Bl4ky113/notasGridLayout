Que es Grid

Grid Layout o solamente Grid, es un sistema de diseño de páginas web orientado a diseño en 2 dimensiones, es decir hacía arriba y abajo, X y Y, entre otros. Esta forma de diseñar esta en un Grid o rejilla, donde se tiene diferentes celdas, columnas y filas. Siendo parecidas a una tabla, pero muchisimo mejor. De hecho Grid viene a ser un excelente o la mejor forma de diseñar sitios web. Pasando de las tablas, los Floats, posicionar diferentes inline-blocks y Flexbox, siendo este orientado mucho a diseños horizontales o de sola una dimensión. Pero esto no significa que Flexbox sea peor que Grid, de hecho se puede hacer grandiosos trabajos con Grid y Flexbox al mismo tiempo, siendo muy utiles y versatiles.

Soporte Web de Grid

Como Grid es una gran implementación en el desarrollo y diseño web, es muy importante que funcione en todos los dispositivos por igual. En general Grid puede funcionar de forma nativa en todos los dispositivos, computadores y celulares, y en la mayoria de navegadores, obviamente excluyendo a IE. Pero si llega a haber un atributo, o valor no soportado sera escrito acá.

Beneficios de Grid

Grid al nos ahorra el trabajo que eran las tablas del pasado en los 2000's, el tener que hacer mil y un conectores para organizar diferentes Widgets y el usar floats o usar Flexbox de una forma no apropiada. Esto nos volvera nuestra vida mucho más sencillo al momentos de hacer un sitio web con una excelente Semantíca (header, main, aside, nav, footer, article, section). Al poder cumplir esto, nos puede dar también una gran ayuda a el momento de hacer un sitio web Responsive a el dispositivo.

Dejando los benefios en esta lista.

  • Metodos muy sencillos
  • Semantica Web Excelente y Organizada
  • Ayuda y Asistencia a el Diseño Responsive

Terminología de Grid Layout

Es muy importante entender, saber y poder usar los terminos sencillos que se usan en Grid Layout, esto para que nosotros podamos entender y saber en donde y cómo estamos haciendo el diseño.

La Teminología de Grid Layout es:

  • Grid Container
  • Grid Item
  • Grid Line
  • Grid Cell
  • Grid Track
  • Grid Area

Grid Container

Es el elemento padre directo de todo del diseño de Grid Layout y sus elementos, este es puesto por el elemento que tenga aplicado como style:

display: grid;

Grid Item

Son los hijos directos del Grid Container, donde los contenidos de estos no son Grid Items. Lo unico necesario es ser hijo directo del Grid Container.

Grid Line

Es la Linea que divide y forma la estructura del Grid Layout, siendo verticales y horizontales (Columns, Rows).

Grid Cell

Es el espacio entre 2 lineas verticales (Columns) y 2 lineas horizontales (Rows).

Grid Track

Es todo el espacio que puede haber entre 2 lineas, verticales o horizontales.

Grid Area

Es el espacio que hay entre varios Grid Cells

Cloumnas y Filas Fracionarias

Para ocupar todo el Grid Container, podemos hacerlo de una forma medio mala, que es usando los %. O podemos usar una nueva medida que introduce Grid Layout, fr o francción. Divide el Grid Container en partes, se divide cuantas veces usemos la unidad, es decir, si la usamos solo una vez, se divide en 1, si la usamos 2 veces, se divide en 2, y así.

También cuentan las fraciones si se usan por ejemplo: 1fr 2fr 1fr. En este caso se dividiria en 5 partes y se da 2 partes a la columna o fila del centro. 

               #fr-test {
                  display: grid;
                 
                  grid-template-columns: 1fr 2fr 1fr;
                  grid-template-rows: 2fr 1fr 2fr;

Quinta Clase

Un factor muy importante al momento de usar Grid para que nuestro diseño sea Responsive, necesitamos usar las Areas.

Areas En Grid Layout

Son una forma de categorizar y posicionar Grid Items, estas Areas las vamos a categorizar mediante nombres y una cuadricula hecha en el StyleSheet Estas areas nos pueden ayudar mucho a hacer un formato de web con header, main, asides, navs, footer y otros.

Para definir que areas queremos dentro del Grid Layout, vamos a especificarlas con grid-template-areas:, pero en areas vamos a introducir los nombres y Grid Cells que va a ocupar cada area, ejemplo: 

               /* Tenemos una cuadricula con 4 columnas y 3 filas */
               grid-template-areas: "header header header header" /* Un header de 4 columnas y una fila */
                                    "aside  main   main   nav" /* Un aside de una columna y una fila, un nav igual y un main con dos columnas. */
                                    "footer footer footer footer" /* Un footer de 4 columnas y una fila */

Ahora con las areas definidas podemos usar grid-area o los mil y un formas sin shortcut que hay para asignar un elemento a esa area. Lo mejor de esto, es que en el html no es necesario tener los elementos en orden, solo un style que los asigne a el area y ya! Ejemplo de Areas. 

               #GridArea_1-test {
                  display: grid;
                 
                  grid-template-rows: repeat(3, 1fr);
                  grid-template-rows: repeat(3, 5em);
                  grid-template-areas: "header header header"
                                       "aside  main   nav"
                                       "footer footer footer";
               }

Sexta Clase

Como hemos visto, la mayoria de veces Grid Layout coge y estira los Grid Items que estan dentro de las Grid Cells. Para evitar que haga esto podemos usar diferentes propiedades para darles un efecto o diseño especifico. Para esto usamos las propiedades de Alineación de Elementos.

Alineación de Elementos en Grid Layout

Para alinear TODOS los elementos del Grid Layout, podemos usar dos principales Propiedades:

justify-items:

Alinea los elementos de una forma horizontal, puede usar algunos valores, como:

  • Start: Alinea a los elementos en el inicio horizontal del Grid Cell, generalmente solo ocupando lo que indica su boxmodel.
  • End: Alinea los elementos en el final horizontal del Grid Cell, generalmente solo ocupando lo que indica su boxmodel.
  • Stretch | Auto: Estira los elementos para que ocupan todo el espacio horizontal del Grid Cell.
  • Center: Alinea los elementos horizontalmente en el centro del Grid Cell, respetando su boxmodel.
align-items:

Alinea los elementos de una forma vertical, puede usar algunos valores, como:

  • Start: Alinea los elementos en el inicio vertical del Grid Cell, generalmente solo ocupando lo que indica su boxmodel.
  • End: Alinea los elementos en el final vertical del Grid Cell, generalmente solo ocupando lo que indica su boxmodel.
  • Stretch | Auto: Estira los elementos para que ocupan todo el espacio vertical del Grid Cell.
  • Center: Alinea los elementos verticalmente en el centro del Grid Cell, respetando su boxmodel.

Pero que tal si queremos que un elemento sea diferente a los demás? un elemento revelde que no le guste seguir las reglas de los demás?

Para eso usaremos en cada Grid Item individualmente (si es necesario o si es un revelde), las siguientes propiedades:

  • justify-self
  • align-self

Estas tienen las mismas propiedades disponibles que justify-items y align-items.

Ejemplo de todas las propiedades con justify-self y align-self. 

               #alignItems-test {
                  display: grid;
                 
                  grid-gap: 20px;
                  grid-template-columns: repeat(2, 1fr);
                  grid-template-rows: repeat(2, 10em);
                 
                  #item-start { justify-self: start; align-self: start; }
                  #item-end { justify-self: end; align-self: end; }
                  #item-stretch { justify-self: auto; align-self: stretch; }
                  #item-center { justify-self: center; align-self: center; }
               }

Septima Clase

Algunas veces en nuestro Grid Container se va a tener un espacio extra en el que podemos usar para alinear los Grid tracks del Grid Layout.

Alineación de los Grid Tracks

Se usan dos propiedades como para alinear los items, pero esta vez enfocado a los Grid Tracks.

Pero ojo, es necesario tener estos espacios, ya que sin estos, ninguna de estas propiedades va a funcionar correctamente.

justify-content:

Se usa para alinear los Grid Tracks de forma horizontal. Se pueden usar los siguientes valores:

  • Start / Normal: Deja los Grid Tracks horizontalmente en el inicio del Grid Container.
  • End: Deja los Grid Tracks horizontalmente en el final del Grid Container.
  • Stretch: Se estrian los Grid Tracks horizontalmente para ocupar todo el Grid Container.
  • Space-Around: Deja un espaciado igual entre cada Grid Track horizontalmente, dejando un la mitad de ese espacio para el final y inicio del Grid Container.
  • Space-Between: Deja los Grid Tracks repartidos horizontalmente entre todo el Grid Container.
  • Space-Evenly: Deja un espaciado igual entre cada Grid Track y el Grid Container horizontalmente.
align-content:

Se usa para alinear los Grid Tracks de forma vertical. Se pueden usar los siguientes valores:

  • Start / Normal: Deja los Grid Tracks verticalmente en el inicio del Grid Container.
  • End: Deja los Grid Tracks verticalmente en el final del Grid Container.
  • Stretch: Se estrian los Grid Tracks verticalmente para ocupar todo el Grid Container.
  • Space-Around: Deja un espaciado igual entre cada Grid Track verticalmente dejando un la mitad de ese espacio para el final y inicio del Grid Container.
  • Space-Between: Deja los Grid Tracks repartidos verticalmente entre todo el Grid Container.
  • Space-Evenly: Deja un espaciado igual entre cada Grid Track y el Grid Container verticalmente.

Octava Clase

En Grid Layout, hemos visto que algunos elementos quedan a la perfección dentro de un Grid o una cuadricula, estas cuadriculas se les conoce como Grid Explicitas. Siembargo, al momento de añadir solo un elemento más a esta cuadricula, puede romper el orden y diseño de la cuadricula. Quedando desorganizado o diferente a el tamaño y propiedades de las columnas y rows del Grid Container. Para evitar esto podemos usar la siguiente propiedad:

Grid-auto-columns/rows:

Con esta propiedad podemos indicar que tamaño van a tener cada nuevo elemento que sea ingresado a un Grid Container, y que este rompa el orden y diseño del Grid Layout. Podemos definir el tamaño de la columna o fila en la que se va a guardar este nuevo elemento. Ejemplo 

               #gridAuto-test {
                  display: grid;
               
                  grid-gap: 1em;
                  grid-template-columns: 1fr 1fr 1fr;
                  grid-template-rows: 5em 3em 5em;
                  grid-auto-rows: 10em; /* Tamaño para los Hellos en las nuevas Rows */
               }

Pero como sé si el elemento se va a añadir en una nueva columna o en una nueva fila? Es algo sencillo, usando otra propiedad podemos cambiar el flujo de los Grid Items, es decir, cambiar la forma en la que se organizan en el Grid Container. Para esto, se usa la propiedad:

grid-auto-flow:

Se usa esta propiedad para determinar el flujo o la forma en que se van a organizar los elementos del Grid o Grid Items. Esto se puede usar solo para organizar los Elementos de una forma vertical o en columnas, o de una forma horizontal o en filas. Para esta propiedad se le puede dar los siguientes valores:

  • row: Organiza los Grid Items en filas, cuanto esta llega a el número total de columnas, se crea otra fila para los demas elementos.
  • column: Organiza los Grid Items en columnas, cuanto esta llega a el número total de filas, se crea otra columna para los demas elementos.
  • dense row/column: Organiza los Grid Items en columnas o en filas pero en caso, que por algun span de cualquier Grid Item, se genere un espacio vacio en el Grid Container, se va a rellenar este con un elemento que pueda entrar ahí. Creando un grid más organizado y menos vacio.

Novena Clase

En el momento de hacer un diseño Responsive debemos asegurar que nuestras columnas y filas sean responsivas también, pero algunas veces 12 columna o filas no va a ser mejor en un diseño de celular, esto puede ser ya que los elementos no se organicen bién o se vean muy muy pequeños. Esto lo podemos hacer con "@media", pero deberíamos ver y intentar con cada tamaño de pantalla, para que sea la cantidad necesaria de columnas y rows.

Para evitar esto, vamos a usar unos valores que se pueden usar en la funcion de repeat() o en el atributo de grid-template-columns/rows, estos valores son:

Auto Fill

Se usa para que la cantidad de columnas o filas, dependiendo del grid-auto-flow, sea automatica a el tamaño de la pantalla, pero esto lo consigue añadiendo más y más columnas a el Grid Container cuando se agranda la pantalla. Cuando se vuelve más pequeña la pantalla, se crea una nueva fila o columna y se organizan ahí los elementos que no entran. Se agrega de la siguiente forma:

grid-template-columns/rows: repeat(auto-fill, <"Tamaño de la columna/fila">)

Ejemplo de autofill, debes usar la consola de desarrollador para ver como cambia con diferentes pantallas. 

               #autofill-test {
                  display: grid;
                 
                  grid-gap: 1em;
                  grid-template-columns: repeat(auto-fill, minmax(2em, 1fr));
               }

Clase Extra | Sub Grid

Al momento de hacer una cuadricula en Grid, en nuestros Grid Items, se pueden ser algo como un articulo, pero estos casí siempre no van a ser iguales. Ya que puede que un titulo o un parrafo de entrada sea más pequeño o grande, esto lo podemos arreglar agregando Sub Grid a estos elementos.

Pero Ojo, esta propiedad a la fecha de hoy 05/02/2021 (mm/dd/aaaa), esta función solo se encuentra disponible para el navegador FireFox

Para usar SubGrid, en cualquier navegador, podemos hacer uso de valorres, columnas, filas iguales que no va a cambiar el resultado de navegador a navegador. Excepto si es IE, nadie quiere a IE.

¿Cómo Usar SubGrid?

Para usar correctamente SubGrid, necesitamos un Grid Container que tenga varios elementos que tengan contenido similares, o que deban ser organizados de una misma forma. Como por ejemplo los articulos de un portal de noticas, por ahora solo vamos a ver cómo son los elementos sencillos. Pero algunas veces estos ariculos tienen un titulo más largo, una introducción más larga, o puede que sus likes, comentarios y demás dañen el orden y espacio de las columnas del articulo, siendo un poco más a la izquierda y a la derecha en diferentes articulos.

Un Ejemplo de donde podemos inicar es: 

               #subgrid_1-test {
                  border: 0.1em solid #ccc;
                  padding: 1em;
                 
                  background-color: #efefef;
                 
                  display: grid;
                  grid-template-columns: repeat(2, 1fr);
                  grid-gap: 1em;
                 
                  .publicacion {
                     background-color: #fefefe;
                     padding: 0.5em;
                 
                     .title {
                        font: 800 1.5em 'Roboto Mono', monospace;
                     }

                     .texto {
                        font: 500 1em 'Roboto', sans-serif;
                     }
                  
                     .data-articulo {
                     
                        button {
                           border: none;
                     
                           background-color: inherit;
                     
                           font: 800 0.8em 'Roboto', sans-serif;
                     
                           span.comentario {
                              text-decoration: underline solid #555 0.2em;
                           }
                           span.likes {
                              text-decoration: underline solid #c00 0.2em;
                           }

                           span.retweets {
                              text-decoration: underline solid #0df 0.2em;
                           }
                        }
                     }
                  }
               }
